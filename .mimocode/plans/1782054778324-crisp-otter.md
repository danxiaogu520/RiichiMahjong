# 交互式测试模块方案

## 目标

创建一个 CLI 交互式测试工具，用户输入手牌/参数，调用 riichi-logic 的纯函数，输出结果供用户验证。

## 可测试模块分析

所有 riichi-logic 的公开函数都是**纯函数**（无副作用、确定性输出），非常适合交互式测试：

| 命令 | 调用函数 | 输入 | 输出 |
|------|---------|------|------|
| `shanten` | `ShantenCalculator::calculate()` | 手牌（13/14张） | 向听数 |
| `win` | `is_winning()` / `is_win_shape()` | 手牌（14张） | 是否和了形 |
| `decompose` | `decompose_hand()` | 手牌（14张） | 所有分解方式 |
| `wait` | `analyze_wait_tiles()` | 手牌（13张） | 听牌列表 + 听牌类型 |
| `dora` | `calculate_dora()` | 全部牌 + 宝牌指示牌 | 宝牌/赤宝牌/里宝牌数 |
| `discard` | `analyze_discard()` | 手牌（14张） | 每张打牌的向听/进张/改良 |
| `acceptance` | `analyze_acceptance()` | 手牌（13张） | 进张/改良/向听 |
| `fu` | `calculate_fu()` | 手牌分解 + 副露 + 条件 | 符数 |
| `points` | `calculate_points()` | 翻/符/庄闲/自摸荣和 | 点数变化 |
| `check` | `check_win()` | 手牌 + WinContext | 和了结果（役种/翻/点） |

## 关键前置：手牌解析器

当前代码**没有** `FromStr` 实现。需要先实现手牌解析：

```
输入格式: "1m 2m 3m 4m 5m 6m 7m 8m 9m 1p 2p 3p 4p"
         "1z 2z 3z 4z 5z 6z 7z"
         "1m1m1m" 或 "1m×3" 表示 3 张相同的牌
```

解析逻辑：
- `1m`-`9m` → 万子 TileType(0-8)
- `1p`-`9p` → 筒子 TileType(9-17)
- `1s`-`9s` → 索子 TileType(18-26)
- `1z`-`4z` → 风牌 TileType(27-30)
- `5z`-`7z` → 三元牌 TileType(31-33)

## 实现方案

### 新建 crate: `crates/riichi-test/`

```
crates/riichi-test/
  Cargo.toml
  src/
    main.rs          — CLI 入口 + REPL 循环
    parse.rs         — 手牌解析（"1m 2m 3m" → Vec<TileType>）
    cmd_shanten.rs   — 向听计算测试
    cmd_analysis.rs  — 和了判定/听牌分析/分解测试
    cmd_dora.rs      — 宝牌计算测试
    cmd_discard.rs   — 打牌分析/进张分析测试
    cmd_scoring.rs   — 符数/点数计算测试
    cmd_check.rs     — 完整和了检查测试
```

### main.rs — REPL 循环

```
> shanten 1m 2m 3m 4m 5m 6m 7m 8m 9m 1p 2p 3p 4p
向听: 0

> win 1m 1m 1m 2m 3m 4m 5m 6m 7m 8m 9m 9m 9m 9m
和了: true

> wait 1m 2m 3m 4m 5m 6m 7m 8m 9m 1p 2p 3p 4p
听牌: 1p(4张) 4p(4张) 7p(4张)
类型: 两面/两面/两面

> dora 1m 2m 3m --indicator 1m
宝牌: 2m × 2

> discard 1m 1m 2m 3m 4m 5m 6m 7m 8m 9m 9m 9m 9m 1m
打 1m: 向听0 进张4种16张
打 9m: 向听0 进张4种16张
...

> points 4 30 0 1 false
闲家荣和 4翻30符: 7700点

> help
可用命令: shanten win decompose wait dora discard acceptance fu points check quit
```

### parse.rs — 手牌解析

```rust
pub fn parse_tile_types(input: &str) -> Result<Vec<TileType>, String>
pub fn parse_tiles(input: &str) -> Result<Vec<Tile>, String>
```

支持格式：
- 空格分隔: `"1m 2m 3m"`
- 连写: `"1m2m3m"`（按 2 字符分割）
- 重复: `"1m×3"` = 3 张 1m

### 依赖

```toml
[dependencies]
riichi-core = { path = "../riichi-core" }
riichi-logic = { path = "../riichi-logic" }
```

## 执行顺序

1. 创建 `crates/riichi-test/` + Cargo.toml
2. 实现 `parse.rs` — 手牌解析器
3. 实现 `main.rs` — REPL 循环 + 命令路由
4. 逐个实现各命令模块
5. 验证：运行 `cargo run -p riichi-test`，手动测试各命令

## 验证

```bash
cargo run -p riichi-test
> shanten 1m 2m 3m 4m 5m 6m 7m 8m 9m 1p 2p 3p 4p
向听: 0    # 用户验证：这是一向听的手牌，正确
> win 1m 1m 1m 2m 3m 4m 5m 6m 7m 8m 9m 9m 9m 9m
和了: true # 用户验证：这是和了形，正确
```
