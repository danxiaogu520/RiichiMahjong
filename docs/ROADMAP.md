# 路线图

## 当前基线

核心牌型、向听/和牌分析、役种与点数计算、引擎行动流程、AI 打牌和本地终端客户端已经形成 workspace。规则仍有部分实现缺口，详见 [`RULES.md`](RULES.md)。

## 优先级

1. 完成西入与 all-last 的终局状态机，并补足结算回归测试。
2. 将响应窗口统一为按优先级、等待超时和多家荣和的确定性流程。
3. 修正四杠散了、四风连打、四家立直和流局满贯的时序。
4. 补齐鸣牌食替和改听后的舍牌振听行为。
6. 为协议增加版本策略和更完整的客户端/服务端集成测试。

## 暂不承诺

本项目当前不承诺联网大厅、持久化账号、段位系统、赛事规则变体或稳定的跨版本网络兼容性。任何新增承诺都应先更新 README、协议文档和测试边界。
