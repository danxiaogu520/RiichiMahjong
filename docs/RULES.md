# 规则口径

本项目固定实现四人立直麻将，不提供运行时规则切换。本文是实现和测试的规则基线；如果代码与本文不一致，应先记录为缺口，再决定是否修改实现。

## 基本设置

- 起始点数 25,000，终局目标 30,000，启用击飞。
- 使用东一至南四；南四未结束时按庄家连庄/过庄进入西场，西一至西四同样执行 all-last。
- 同点按东一局开始时的固定座次牌序排名。
- 不计算段位分、马点、返点或其他额外排名分。

## 和了与役种

启用立直、双立直、一发、门清自摸、平和、七对子、国士无双、天和、地和、岭上开花、海底/河底、抢杠和，以及标准役种和役满。宝牌、赤宝牌和里宝牌不能单独起和；役满成立后不再计算普通役和宝牌，复合役满按倍数叠加。

启用食断、舍牌振听、同巡振听、立直振听和立直后暗杠限制。允许后付/片和：最终和牌分解有役即可。

## 杠、鸣牌和途中流局

- 赤宝牌为万、筒、索各一张。
- 杠成立后增加宝牌指示牌；暗杠立即翻牌，大明杠立即翻牌，加杠在抢杠窗口结束且成立后翻牌。
- 吃、碰、杠会取消一发；立直后只能暗杠，且前后听牌牌种集合必须相同。
- 九种九牌、四风连打、四家立直、四杠散了均须先处理其优先级更高的荣和/抢杠/岭上流程，再确认流局。
- 流局满贯按满贯自摸支付，不听罚符不与之叠加，立直棒留在场上。

## 响应与结算顺序

响应优先级固定为：`荣和 > 大明杠 > 碰 > 吃`。响应不按网络到达顺序处理；同一优先级的合法荣和需等待所有具备资格的玩家响应或超时，多家荣和同时成立，头跳按固定座次距离确定。

所有结束原因统一遵循：

```text
RoundEnd → 应用全部点数变化 → 结算排名 → 检查终局 → 进入下一局
```

终局时剩余立直棒交给结算排名第一名，加入后不重新排名。

## 实现缺口

当前代码仍需持续校正的重点包括西入流程、完整同步响应等待、四杠散了的后续窗口、流局满贯与改听解除舍牌振听。它们是开发任务，不应在调用方通过临时分支绕过。
